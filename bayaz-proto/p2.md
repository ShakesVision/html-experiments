Build a VERY SIMPLE browser-based writing tool that visually looks like writing inside an open book.

The goal is simplicity and minimal JavaScript.

Only implement the essential behavior.

The output must be a SINGLE FILE:

index.html

No frameworks.
No build tools.
Use plain HTML, minimal CSS, and minimal JavaScript.

---

CORE IDEA

The screen should display TWO book pages side-by-side like an open novel.

Example layout:

[ Page 1 ] [ Page 2 ]

Each page must look like a blank paper page.

Include:

• page border
• page padding (margins)
• header area at the top of each page
• main writing area below the header

No fancy styling is required.

Just simple CSS to visually resemble book pages.

---

HEADER

Each page must have a simple header row.

Example layout:

Left side: chapter name
Right side: novel title

Example:

Chapter 1 My Novel

This header text can be static placeholders.

Do not build complex header systems.

---

WRITING AREA

Below the header is the main text editor.

Use:

contenteditable

The user types normally.

The editor must support RTL text.

Set default direction to RTL.

---

PAGINATION BEHAVIOR

This is the only important logic.

The editor should behave like pages of a book.

RULE:

When text fills the first page and overflows,
the extra content automatically moves to the second page.

When the second page fills,
new pages are created internally.

However:

ONLY TWO PAGES should be visible at a time.

As the user types,
the content flows forward automatically.

Implementation can be simple:

Maintain a hidden container with the full content.
Measure page height.
Split content into pages based on overflow.

Render the first two pages from that content.

The pagination does not need to be perfect.

Simple overflow splitting is enough.

---

NAVIGATION

Add simple navigation controls below the pages:

Previous button
Next button
Current page indicator

Example:

< 1-2 >

Clicking next shows the next pair of pages.

Example:

Pages 3–4
Pages 5–6

---

CODE CONSTRAINTS

Keep the code extremely simple.

Goals:

• minimal JavaScript
• readable code
• no libraries
• no complicated editor features

Do NOT implement:

toolbars
formatting buttons
fonts
export
printing
autosave
chapters detection
custom headers
font loaders

Those are intentionally removed.

---

STYLING

Minimal CSS only.

Pages should have:

white background
light gray border
padding like a book margin
slight gap between pages

No need for Tailwind.

Simple CSS is enough.

---

FINAL RESULT

The result should be a very lightweight "book-style writing pad":

Two pages visible.
Text flows between pages.
User can navigate page pairs.

Keep the code small, clear, and easy to understand.
